# Meeting Minutes（議事録スキル）

MTGの結果を整理し、研究ノート・プロジェクト管理・タスクリストを連携して更新するスキル。

## 原則

- **当日メモがベース**: 議事録は当日メモの構造を維持し、文字起こしやNotionで強化する
- **要約開始前の完全性確認**: すべてのNotion URLがreferenceに揃ってから要約を開始
- **1on1発話者判定**: 文字起こしは常にユーザーのラベルのみだが、実際は教授との対話を考慮
- **Single Source of Truth**: ローカルMDファイルが正式記録、Notionは参照のみ

---

## Workflow

### **Phase 1: 初期確認とRawファイル作成**

#### 1.1 プロジェクト情報の確認

ユーザーが「MTG終わった」「議事録作って」と言ったら、以下を確認する：

```markdown
お疲れ様でした！議事録の整理を開始しますね。

まず、今回のMTGに関する情報を教えてください：

1. **プロジェクト名**: どのプロジェクトに関するMTGですか？（例: lora-dynamic-separation）
2. **日付**: いつのMTGですか？（デフォルト: 今日 YYYY-MM-DD）
3. **参加者**: 誰が参加しましたか？（例: 教授, ユーザー）
```

#### 1.2 コンテキスト自動読み込み（内部処理）

プロジェクト名が確定したら、**自動的に以下を読み込む**（ユーザーに見せない内部処理）：

**必須読み込み**:
- `lab/projects/<project>/README.md` - プロジェクト概要・現在の状況

**オプション（存在すれば）**:
- `lab/projects/<project>/specs/*.md` - 最新1件（要件定義）
- `lab/projects/<project>/experiments/*.md` - 最新2件（実験結果）
- `lab/projects/<project>/meetings/*.md` - 最新1件（前回のMTG）

これらの情報は議事録生成時に文脈を理解するために使用する。

#### 1.3 Rawファイルの作成と貼り付け依頼

**スキルがRawファイルを作成**し、ユーザーに貼り付けを依頼する：

1. `Write`ツールで空の`lab/projects/<project>/meetings/YYYY-MM-DD-mtg-raw.md`を作成
2. ユーザーに以下のメッセージを送る：

```markdown
コンテキストを読み込みました！

### 📄 ステップ1: 文字起こしの貼り付け

30分の会話データがあるため、ファイル経由でお願いします。

**以下のファイルを作成しました**:
[lab/projects/<project>/meetings/YYYY-MM-DD-mtg-raw.md](絶対パス)

こちらのファイルに**文字起こしの全文を貼り付けて保存**してください。

---

### 📝 ステップ2: 当日メモの貼り付け

文字起こしの貼り付けが完了したら、**このチャットに「完了」とお知らせいただき、同時に当日メモを貼り付けてください**。

**当日メモのフォーマット例**:
\```markdown
# やったこと
- ...

# 次やること
- ...

# 話したこと
- ...

---
- Slack要約

## AI要約
---
\```

準備ができたら、「完了」+ 当日メモをお願いします！
```

---

### **Phase 2: 当日メモの解析とNotion URLの確認**

#### 2.1 当日メモの受信

ユーザーが「完了」+ 当日メモを貼り付けたら：

1. **Rawファイルを`Read`で確認**（文字起こしが貼り付けられているか）
2. **当日メモからNotion URLを抽出**
   - 正規表現: `https://www\.notion\.so/[a-zA-Z0-9\-?=&]+`
   - 例: `https://www.notion.so/3206a9cc58698004b45ef0b70e5570e3?pvs=21`

#### 2.2 Referenceとの照合

**プロジェクトのreferencesフォルダをチェック**:
```python
Glob("lab/projects/<project>/references/notion-*.md")
```

抽出したNotion URLと既存のReferenceを照合：

**ケースA: すべてのURLがReferenceに存在**
→ **要約開始**（Phase 3へ）

**ケースB: ReferenceにないURLが存在**
→ **要約前にユーザーに要求**

```markdown
当日メモ内に以下のNotion URLが含まれていますが、Referenceにありません：

- https://www.notion.so/3206a9cc58698004b45ef0b70e5570e3?pvs=21
- https://www.notion.so/0-9999-1-32a6a9cc5869800da6b5d68dfac3dd37?pvs=21

これらのページを **Markdownでエクスポート**し、以下のパスに保存してください：

- `lab/projects/<project>/references/notion-3206a9cc58698004b45ef0b70e5570e3.md`
- `lab/projects/<project>/references/notion-32a6a9cc5869800da6b5d68dfac3dd37.md`

保存が完了したら「完了」とお知らせください。
```

ユーザーが「完了」と言ったら：
- **Globで再確認**
- すべて揃ったら **要約開始**（Phase 3へ）

---

### **Phase 3: 議事録生成（当日メモベース）**

#### 3.1 情報の統合

**優先順位**:
1. **当日メモ**（ベース構造）
2. **文字起こし**（詳細な発言・議論の流れを補完）
3. **Notionメモ**（背景・詳細な分析を補完）
4. **既存コンテキスト**（README, specs, experiments, meetings）

**生成方針**:
- 当日メモの構造を維持（「やったこと」「次やること」「話したこと」）
- 文字起こしから**具体的な発言や議論の流れ**を抽出し、当日メモの各セクションに追加
- Notionメモから**背景・詳細な考察**を引用し、補足情報として追加
- 既存コンテキストから**前回の決定事項や実験結果**を参照し、文脈を明確化

#### 3.2 1on1発話者判定

**重要**: 文字起こしは常にユーザーのラベルのみ。実際は教授との1on1対話。

**判定ルール**:
- **質問形式**: 「〇〇についてはどう思いますか？」「〇〇はどうでしょう？」→ **教授**
- **回答形式**: 「はい、〇〇を試しました」「〇〇ができました」→ **ユーザー**
- **提案形式**: 「〇〇してはどうか」「〇〇を考えてみて」→ **教授**
- **報告形式**: 「〇〇を実装しました」「〇〇の結果は△△でした」→ **ユーザー**

会話の文脈から発話者を推測し、議事録に反映する。

#### 3.3 統合ファイルの構成

**ファイルパス**: `lab/projects/<project>/meetings/YYYY-MM-DD-mtg.md`

**フォーマット** (当日メモベース):

```markdown
---
date: YYYY-MM-DD
project: <project-name>
participants: [教授, ユーザー]
tags: [meeting, <プロジェクト名>]
---

#### 3.4 生成のポイント

**当日メモの各セクションを強化する方法**:

**「やったこと」セクション**:
- 当日メモの箇条書きを見出しとして使用
- 文字起こしから具体的な成果・実装内容を補完
- Notionメモから詳細な背景や分析結果を引用
- 関連する実験ログやSpecsへのリンクを追加

**「話したこと」セクション**:
- 当日メモのトピックを見出しとして使用
- 文字起こしから「ユーザー報告」と「教授フィードバック」を抽出（発話者判定を適用）
- 決定事項を明確に記載
- Notionメモから背景・補足情報を追加

**「次やること」セクション**:
- 当日メモの箇条書きをそのまま使用
- チェックボックス形式に変換
- 担当者を明記（ユーザー/教授）

### **Phase 4: Rawファイルの削除**

統合ファイル作成後、ユーザーに削除の確認を取る：

```markdown
統合議事録を作成しました！

以下のファイルは削除してもよろしいですか？
（統合ファイルに全文が含まれています）

- `lab/projects/<project>/meetings/YYYY-MM-DD-mtg-raw.md`

削除する場合は「削除」とお知らせください。
```

ユーザーが承認したら、Bashツールで削除する。

**注意**: 当日メモは削除しない（チャットに貼り付けただけなので、ファイルとして存在しない）

---

### **Phase 5: TODO反映**

#### 5.1 タスク選択（ユーザー判断必須）

当日メモの「# 次やること」セクションから抽出したタスクをユーザーに提示：

```markdown
## 📝 抽出されたNEXT ACTIONS

以下のタスクを抽出しました：

### 実験系
1. [ ] 損失を1にする手法を探す
2. [ ] 損失が1じゃなくても分離が意味がある手法を探す（エゴクロス）
3. [ ] 継続学習で可視化をプロット

### 理論系
4. [ ] 評価指標のベクトル化

### その他
5. [ ] AIエージェントで研究活動する具体的な方法を考える

---

**質問**: 以上のタスクから、どれを本日のTODOリスト（`secretary/todos/YYYY-MM-DD.md`）に反映しますか？

- 優先度（最優先/通常/低）もお知らせください。
- 期限がある場合は指定してください。
- 追加・削除・調整も可能です。
```

#### 5.2 TODOリストへの追記

ユーザーが選択したタスクのみを`secretary/todos/YYYY-MM-DD.md`に追記：

**フォーマット**:
```markdown
## 最優先
- [ ] 評価指標のベクトル化 | 優先度: 最優先 | 期限: YYYY-MM-30

## 通常
- [ ] 継続学習で可視化をプロット | 優先度: 通常
- [ ] 損失を1にする手法を探す | 優先度: 通常
```

**重要**: 期限は**明示的に指示がない限り付けない**

---

### **Phase 6: プロジェクト管理の更新**

#### 6.1 README.mdのfrontmatter更新

`lab/projects/<project>/README.md`のfrontmatterを更新：

```yaml
---
project: lora-dynamic-separation
status: in-progress
department: lab
summary: 評価指標のベクトル化フェーズ（スカラじゃない空間版指標を検討中）- MTG YYYY-MM-DD
created: YYYY-MM-DD
last_updated: YYYY-MM-DD
---
```

**更新ポイント**:
- `summary`: 現在の状況を1行で簡潔に記述（MTGの主要な決定事項を反映）
- `last_updated`: 今日の日付

#### 6.2 マイルストーンの更新

README.mdの「## マイルストーン」セクションに新しいタスクを追加：

```markdown
## マイルストーン

### Phase 3: 評価指標の開発
- [x] 加重コーダル距離の限界を確認（YYYY-MM-DD）
- [ ] スカラじゃないベクトル化評価指標を開発 ← **NEW**
- [ ] IOUの空間版のような指標を実装 ← **NEW**
- [ ] 継続学習で可視化をプロット ← **NEW**
```

**注意**: 既存のマイルストーンを上書きせず、新しいタスクを追加する形で更新する。

## ベストプラクティス

### Notion Referenceの管理

- Notion参照は`lab/projects/<project>/references/`に保存
- ファイル名: `notion-<ページID>.md`
- プロジェクトに紐づく情報として一元管理
- 議事録から相対パス`../references/notion-xxx.md`でリンク可能

### 文字起こしの扱い

- 30分の会話データは必ずファイル経由で扱う（チャットに貼り付けない）
- Rawファイルは統合後に削除（統合ファイルがSSOT）

### 当日メモの活用

- 当日メモの構造を最大限尊重する
- ユーザーが重要だと思った内容が反映されているため
- 文字起こしやNotionはあくまで「補完」として使う

### エラーハンドリング

- Rawファイルが空の場合: ユーザーに再度貼り付けを依頼
- Notion URLが抽出できない場合: Skip（Notionメモがない場合もある）
- プロジェクト名が不明な場合: 必ず確認してから進む

---

## 注意事項

- **TODOへの期限は明示的に指示がない限り付けない**
- **既存のマイルストーンを上書きしない**（追加のみ）
- **文字起こしの発話者判定は必ず行う**（1on1の文脈を考慮）
- **すべてのNotion URLがreferenceに揃うまで要約を開始しない**
