## 研究背景

### 解決したい課題
- 複数のLoRA（タスク1, タスク2）の部分空間が「どの成分同士が離れていて、どの成分が近いか」をスカラ指標（加重コーダル距離）だけでは捉えきれない
- 教授提案: **IoUの空間版** = ヒートマップで個別のベクトルペアごとの角度を可視化

### 教授の期待
- 対角（同順位ペア）が赤（離れている） → 重要成分が分離できている ✅
- 非対角（異順位間）が青（近い） → 期待通りの挙動 ✅

---

## 実装仕様（確定版）

### 入力
- `B_1`: タスク1のLoRA重み行列 (n × d)
- `B_2`: タスク2のLoRA重み行列 (n × d)

### 処理フロー

1. **SVD分解で列空間の基底ベクトルを取得**
   ```python
   U_1, Σ_1, V_1^T = SVD(B_1)
   U_2, Σ_2, V_2^T = SVD(B_2)
   ```
   - `U_1`, `U_2` の列ベクトル = 列空間の正規直交基底
   - 特異値の大きい順に並んでいる
   - `U_1[:, 0]` = タスク1の「特異値最大の基底ベクトル」

2. **全ペアの角度行列を計算**
   ```python
   for i in range(k):
       for j in range(k):
           dot = np.dot(U_1[:, i], U_2[:, j])
           angle_matrix[i, j] = np.arccos(np.abs(dot))  # [rad]
   ```
   - 絶対値を取る理由: ベクトルの符号は角度に無関係

3. **ヒートマップ表示**
   ```
                Task2_σ1  Task2_σ2  Task2_σ3  ...
   Task1_σ1        θ11       θ12       θ13
   Task1_σ2        θ21       θ22       θ23
   Task1_σ3        θ31       θ32       θ33
   ...
   ```
   - カラーマップ: `RdYlBu_r` (赤=90°, 青=0°)
   - 対角成分が赤 → 分離成功
   - 非対角成分が青 → 期待通り

## 既存コードとの関係

### `weighted_chordal.ipynb` との違い

既存実装:
- **主角度**（canonical angles）を計算
- より複雑な計算（交差積行列 C = Q_A^T Q_B のSVD）
- 結果は対角成分のみ（k次元ベクトル）

今回の実装:
- **全ペアの角度**を計算（k×k行列）
- よりシンプル（SVDの左特異ベクトルの内積）
- 非対角成分も含めて可視化

→ 既存コードは**流用せず**、新規に実装する方が明確

## 期待される成果

### 可視化から分かること

**理想的なケース（λ=10000, 損失1.0）**:
```
対角: [85°, 82°, 78°, ...]  # 赤い
非対角: [5°, 3°, 8°, ...]   # 青い
```
→ 重要成分が完全に分離している

**問題のあるケース（Dual Optimizer, 損失0.9999）**:
```
対角: [45°, 50°, 40°, ...]  # 黄色〜オレンジ
非対角: [30°, 25°, 35°, ...] # やや青い
```
→ 対角と非対角の差が小さい = 分離不十分

### 加重コーダル距離との対応

- 加重コーダル距離 = 対角成分の加重平均（特異値で重み付け）
- ヒートマップ = 個別成分の詳細を保持
- **相補的な関係**: 両方使うことで完全な理解が得られる

## 実装上の注意点

1. **数値安定性**
   - `arccos` の引数は [-1, 1] にクリップ
   - 絶対値を取る（符号は無関係）

2. **軸ラベルの明確化**
   - 「Task1_σ1」= タスク1の第1特異値に対応する基底
   - 「主ベクトル」ではなく「左特異ベクトル」が正確

3. **カラーマップの選択**
   - `RdYlBu_r`: 赤（90°）→ 黄（45°）→ 青（0°）
   - 対角が目立つように

4. **チェックポイント互換性**
   - PyTorchの `.pth` ファイル形式を想定
   - `state_dict` からLoRA重みを抽出する方法を確認
