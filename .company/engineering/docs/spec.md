# Workout Tracker — Feature Specification

> **Version:** 2026-03  
> **Platform:** Web App (Next.js / PWA)  
> **Data Storage:** `localStorage` (key: `workout_tracker_data`)

---

## 1. Data Model

### 1.1 Region
部位グループ（例: Chest, Back, Legs）。

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | `string` | 一意ID (Date.now文字列) |
| `name` | `string` | 部位名 |
| `color` | `string?` | テーマカラーキー（gray / red / blue / green / yellow / purple / orange） |

### 1.2 Exercise
種目。必ずひとつの Region に属する。

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | `string` | 一意ID |
| `name` | `string` | 種目名 |
| `regionId` | `string` | 所属 Region の ID |
| `memo` | `string?` | 自由記述メモ（種目に紐づく） |

### 1.3 WorkoutLog
特定の日付×種目の記録。

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | `string` | 一意ID |
| `date` | `string` | `YYYY-MM-DD` 形式 |
| `exerciseId` | `string` | 対象種目の ID |
| `sets` | `WorkoutSet[]` | セット一覧 |

### 1.4 WorkoutSet

| フィールド | 型 | 説明 |
|---|---|---|
| `setNumber` | `number` | セット番号（1始まり） |
| `weight` | `number` | 重量 (kg) |
| `reps` | `number` | レップ数 |

### デフォルトデータ
初回起動時に以下の Region / Exercise がプリセットされる（ログは空）。

| Region | Exercise |
|---|---|
| Chest | Bench Press, Incline Dumbbell Press |
| Back | Pull-up, Barbell Row |
| Legs | Squat, Leg Press |
| Shoulders | Overhead Press |
| Arms | Bicep Curl, Tricep Extension |

## 3. ホーム画面 (`/`)

### 3.1 種目リスト表示
- Region ごとにアコーディオン形式でまとめて表示。
- 各 Region はヘッダーをタップして展開／折りたたみ可能。
- ヘッダーに「最終トレーニング日」バッジを表示（Today / Yesterday / Last N days ago / No data）。
- 各 Exercise にも同様の最終記録バッジを表示。

### 3.2 カラーテーマ
Region ごとに 7 色（gray / red / blue / green / yellow / purple / orange）のカラーテーマが設定可能。背景・ボーダー・テキスト・アイコンにテーマ色が適用される。

### 3.3 編集モード
右上の `Edit` ボタンで編集モードを開閉。

- **Region の追加:** 「Add custom body part」を押して名称を入力 → 追加。
- **Region の名称変更:** インライン入力で編集。
- **Region のカラー変更:** 編集モード時にヘッダー下にカラーピッカーが展開される。
- **Region の削除:** ゴミ箱アイコン → 確認ダイアログ → 削除（関連する Exercise・WorkoutLog もすべて同時削除）。
- **Exercise の追加:** 「Add custom exercise」を押して名称を入力 → 追加。
- **Exercise の名称変更:** インライン入力で編集。
- **Exercise の削除:** ゴミ箱アイコン → 確認ダイアログ → 削除（関連する WorkoutLog も削除）。

### 3.4 ドラッグ＆ドロップ並び替え
`@dnd-kit` ライブラリを使用。編集モード中、Region・Exercise それぞれの左端に表示されるグリップアイコンをドラッグして並び替えが可能。

## 4. カレンダー画面 (`/calendar`)

### 4.1 カレンダー表示
- 月表示 / 週表示を右上の切り替えボタンで選択可能。
- デフォルト表示モードはアプリ設定で変更可能。
- 前後のナビゲーション（←／→）で月・週を移動。

### 4.2 日付セルのビジュアライズ
- トレーニング記録がある日は、各 Region の色ドット積み上げバーをセル内に表示。
- バーの高さはセット数に比例（最大 15 セットで 100% 高さ）。
- 今日の日付はハイライト表示。
- 記録のない日はタップ不可（disabled）。

### 4.3 サマリー統計カード
カレンダー上部に以下の 2 つのカードを表示：
- **Days This Month:** 今月のトレーニング日数（ユニーク日付数）
- **Days This Week:** 今週のトレーニング日数

### 4.4 カラーレジェンド
カレンダー下部に全 Region のカラーとラベルを一覧表示。

### 4.5 日別詳細モーダル
記録のある日付をタップすると、画面下から（PCでは中央に）モーダルがスライドアップ。

- スクロール可能なリストで、その日に記録されたすべての種目が表示される。
- 各種目カードは**左半分（種目名＋Max重量）**と**右半分（展開アイコン）**で分割されており：
  - **左半分をタップ:** 種目詳細画面（`/exercise/[id]?date=日付`）に遷移。
  - **右半分の矢印をタップ:** アコーディオン展開でセット一覧を表示。
- 展開時のセット表示は読み取り専用コンパクトグリッド（後述、§6参照）。

## 5. 種目詳細画面 (`/exercise/[id]`)

### 5.1 ヘッダー
- 左側に戻るボタン（`<`） と、現在閲覧中の Region 名（カラーバッジ付き）、種目名を大きく表示。
- 右側に経過タイマー表示（開始時刻からの経過時間を `mm:ss` 形式でリアルタイム更新）。

### 5.2 日付切り替え
種目名下のドロップダウン風テキスト（`Today's Sets` / `M/D's Sets`）をタップするとネイティブの日付ピッカーが開き、過去の日付のデータを閲覧・編集可能。

### 5.3 Memo（メモ）
- 種目に紐づいた自由記述メモを保存・表示する。
- 未編集時: `MEMO` アイコン + プレビューテキストが折りたたまれた状態で表示。
- タップで編集モードに展開：
  - テキストエリアは入力内容に合わせて**自動的に高さが広がる**（スクロールバーなし）。
  - 展開時のカーソルは**テキスト末尾**に自動配置される。
  - `onBlur` でメモを自動保存。
  - 「Save」「Cancel」ボタンで明示的に操作も可能。

### 5.4 Last Session カード
直近の過去セッションを 1 件表示する折りたたみカード。

- **折りたたみ時（ヘッダー）:**
  - 左: `LAST SESSION (M/D)` ラベル（今年は `M/D`、過去年は `YYYY/M/D`）
  - 中央（絶対配置でカード横幅の真ん中）: `Max ●●kg` （最大重量）
  - 右: 展開アイコン（`▼`）
- **展開時:** 読み取り専用コンパクトグリッドでセット一覧を表示（§6参照）。

### 5.5 Today's Sets（セット入力エリア）

#### タイトル
- 選択日が「今日」の場合: `Today's Sets`
- 過去の日付の場合: `M/D's Sets`
- タイトルをタップすると日付ピッカーが開く（§5.2と同じ。）

#### セット行のUI
各セット行は角丸コンテナ(`rounded-2xl`)で表示。

| 列 | 内容 |
|---|---|
| 左端（丸バッジ） | セット番号（circle badge） |
| WEIGHT 欄 | 数値入力（中央揃え）+ 右隣に `kg`（動的追従） |
| 中央区切り | `×`（薄いグレー） |
| REPS 欄 | 数値入力（中央揃え） |

- **スマートフォーカス:** 入力欄をタップすると現在の数値が自動で全選択される（即時上書き入力）。
- **スピンボタン非表示:** ブラウザ標準の up/down 矢印 UI を非表示にし、数値の完全中央揃えを実現。
- **`kg` の動的追従:** 数値の右隣に `kg` を固定オフセットではなく、数値のテキスト幅に追従して配置。桁数が変わっても数字と埋もれない。
- **スワイプ削除（Swipe to Delete）:** 行を左にスワイプすると赤い削除ボタンがスライドイン。ゴミ箱アイコン + `Delete` ラベル。閾値は 40px で「スナップオープン」する。行外タップで閉じる。

#### セットの追加・保存
- `+ Add Set` ボタンで新しいセット行を追加。
- `Save Workout` ボタンで全セットを保存。
  - 保存成功後ボタンが `Saved!` (緑) にアニメーション変化。
  - 全セットを削除した状態で保存すると記録が除去され、ボタンが `Deleted!` (赤) に変化。

### 5.6 Progress グラフ
過去のトレーニングデータをラインチャートで可視化（Recharts 使用）。

- **期間フィルター（1M / 3M / ALL）:** グラフ右上のボタンで表示期間を切り替え。
  - `1M`: 直近 1 ヶ月
  - `3M`: 直近 3 ヶ月
  - `ALL`: 全期間
- **軸の日付フォーマット:** `M/D`（今年）または `YYYY/M/D`（過去年）で統一表示。
- **グラフタップで拡大モーダル:** グラフエリアをタップすると全画面モーダルで拡大表示。モーダル内でも期間フィルターを操作可能。インタラクティブ Tooltip 付き。
- **History ボタン:** グラフカード右下に独立した `History` ボタン。タップで「過去セッション一覧モーダル」を開く。モーダルから日付を選択すると、当該日のデータを編集可能。

## 6. セット表示の共通 UI

**対象:**
1. Last Session カード（展開時）
2. カレンダー詳細モーダルの各種目（展開時）

**デザイン:**
- 入力フィールド不使用。枠線・背景なし。完全読み取り専用。
- ヘッダー行: `Weight` | `(空)` | `Reps`（12px、uppercase、グレー）
- 各セット行:

| 列 | 内容 |
|---|---|
| 左端（丸バッジ） | セット番号（小さな circle badge） |
| WEIGHT 欄 | 数値 + 右隣に `kg`（数値幅に動的追従、`ml-2` の隙間） |
| 中央区切り | `×`（薄いグレー） |
| REPS 欄 | 数値 |

## 9. データ永続化

- すべてのデータは `localStorage` に JSON 文字列として保存（キー: `workout_tracker_data`）。
- アプリ起動時に読み込み。パースエラー時はデフォルトデータにフォールバック。
- データ変更（Region・Exercise の追加・削除・並び替え、WorkoutLog の保存）のたびに `localStorage` を同期更新。
- **カスケード削除:** Region 削除時は傘下の Exercise および WorkoutLog もすべて同時削除される。Exercise 削除時も同様に WorkoutLog が削除される。

## 12. 新アーキテクチャ要件 (2026-03-22 策定 / 03-28 実装開始)

👑 **【Workout Tracker 新アーキテクチャ要件定義書（最終版）】**

### 12.1 アプリ全体の画面構成とデータ連携の大原則
**4タブ構成**: ボトムナビゲーションを左から Home, Exercises, Calendar, Config の順で配置する。

**役割の明確化**:
- **Exercises（旧Home）**: すべての種目を部位ごとに管理する「マスター図鑑」。
- **Home（新設）**: 日々のトレーニングメニュー（ルーティン）を構築・管理・実行する画面。

**データ設計（リレーショナル）**: Home に保存される種目データは、Exercises の「種目ID」の参照のみとする。
**カスケード削除**: Exercises で種目が削除された場合、ルーティン（Home）側からも自動的に見えなくなる。

### 12.2 Home 画面の基本UIと編集モード
- Exercises 画面と全く同じ位置・同じUIの「編集モード」ボタンを配置する。
- **編集モード時の追加アクション**:
  - Exercises の部位追加ボタンと全く同じデザイン・配置の大きな追加ボタンを設置. テキストは `[＋ Add Routine or Rest]` とする。
  - タップすると画面下部からアクションシートが開き、「Routineを追加」「Restを追加」を安全に選択できる。

### 12.3 各カード（コンテナ）の仕様
**Routine（ルーティン）カード**:
- Exercises の部位カードと完全に同じ丸角コンテナUI。展開可能。
- 既存の部位カラーとは明確に異なる、独立した専用カラーを持つ。
- Routineカード自体には「最終トレーニング日」バッジは表示しない。
- **追加ボタン**: 編集モード時のみ、展開した最下部に `[＋ Add Exercise]` ボタン（旧Homeと同じデザイン）を表示する。

**Routine内のExerciseカード**:
- 旧Homeのエクササイズカードと全く同じ見た目。元の部位のテーマカラーを継承する。
- 「最終トレーニング日」バッジを表示する。
- タップで詳細画面へ遷移する挙動も維持する。

**Rest（休み）カード**:
- 縦幅を極限まで圧縮した丸角デザイン。タップや展開機能を持たない「純粋な区切り線（セパレーター）」。

### 12.4 ルーティンと種目の操作・管理（編集モード時）
- **Routine / Rest 自体の管理**: 「並び替え」および「削除」が可能。
- **Routine内のExerciseカードの管理（完全踏襲）**:
  - 「並び替え」および「削除」が可能。
  - UIおよび挙動は、旧Home（現Exercises）ですでに実装されているものと全く同じにする。
  - ※ここでの削除は「ルーティンからの除外」であり、マスターデータは消えない。

### 12.5 [＋ Add Exercise] 押下時の種目選択UI（モーダル）
- **UIのハイブリッド再利用**: 既存の「カレンダー日別詳細モーダル」の実装（オーバーレイ等）を再利用しつつ、中身のアコーディオンは Exercises 画面（旧Home）と全く同じUIを踏襲する。不要な機能（履歴等）は全排除。
- **重複禁止と「追加済み」UI**: 同一種目の重複追加は不可。すでにRoutineにある種目はチェックボックスをグレーアウト（タップ不可）にする。
- **追加専用（Append）挙動**: 開くたびに選択状態はリセットされる。
- **選択/確定アクション**: タップ順に「数字（1, 2, 3...）」が振られるチェックボックスを採用。下部の確定ボタンで、選択順通りにRoutineの一番下へ追加する。

### 12.6 [ ▶ Start Workout ] 押下時の実行フローとUI
- **トリガー**: Routineカードのヘッダー部分に配置した `[ ▶ Start Workout ]` ボタンを押して開始。
- **画面遷移とUI（ウィザード形式）**:
  - 既存の「種目詳細画面」を再利用する。
  - 詳細画面のスクロールに追従する既存ヘッダーのすぐ下に、ウィザード用の特殊ヘッダー（「前の種目へ」「次の種目へ」ボタン）を追加表示する。
- **ナビゲーションの挙動（Replace）**:
  - 「次へ/前へ」で移動する際は、画面を上に積む（Push）のではなく、現在の画面を「切り替える（Replace）」仕様とし、戻る履歴を汚さないベストプラクティスを採用する。
- **中断（Quit）と状態保持**:
  - 専用の終了ボタンは設けず、画面左上の既存の「戻る」ボタンを押した場合はHome画面へ戻る（中断扱い）。
  - その際、Home画面のRoutineカードのアコーディオンの開閉状態は、開始前と全く同じ状態を保持しておく。
- **トレーニング終了と演出**:
  - 最後の種目の詳細画面では、「次の種目へ」ボタンが `[ トレーニング終了 ]` ボタンに変わる。
  - タップするとHome画面に戻り、既存の「Saved!」と同じ控えめなデザインで「Workout Complete!」等のトースト通知を表示する。
