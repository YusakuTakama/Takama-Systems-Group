# Google Calendar MCP 連携プラン

## 概要
オーナーの Google Calendar と AI (Antigravity) を Model Context Protocol (MCP) で連携し、スケジュール管理の効率化を図る。

## フェーズ1：閲覧・把握（現在の目標）
- **目的**: 秘書（AI）がオーナーの予定を知り、状況に合わせた提案ができるようにする。
- **具体的内容**:
  - `list_events`, `get_event` 等による予定の読み取り。
  - カレンダーの空き時間をチェックし、TODOの実行タイミングを提案する。
  - 重複した予定や、タイトすぎるスケジュールへのアラート。

## フェーズ2：自動操作（将来の目標）
- **目的**: チャットまたは自動化によるスケジュール入力を可能にする。
- **具体的内容**:
  - `create_event`, `update_event`, `delete_event` による操作。
  - TODOリストの「高優先度」タスクを自動でカレンダーの枠に確保。

## 検討事項
- **セキュリティ**: どの Google アカウントを連携するか（大学用 / 個人用）。
- **MCPサーバー**: Google 公式の `google-maps` や `google-calendar` MCP サーバーの利用を検討。

## 他の PC での利用について

MCP の設定は PC ごとに必要です。別の PC でもカレンダーを使えるようにするには、以下の手順を行ってください：

1. **設定のコピー**:
   - この PC の `mcp_config.json` の内容を、新しい PC の Antigravity 設定に貼り付けます。
2. **パスの確認**:
   - `GOOGLE_OAUTH_CREDENTIALS` のパスが、新しい PC の OneDrive パスと一致しているか確認してください。
3. **認証の再実行**:
   - 新しい PC のターミナルで `npx -y @sudomcp/google-calendar-mcp auth` を実行し、再度ブラウザで許可を与えてください。
   - ※認証トークンは PC ごとのローカルフォルダに保存されるため、この作業が必要です。
