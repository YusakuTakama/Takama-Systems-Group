# LoRA部分空間可視化システム：構成メモ (2026-03-26)

## 1. システム構成 (Modularization)
コードのメンテナンス性と再利用性を高めるため、機能を2つのモジュールに分離しました。

| モジュール | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **コア・ユーティリティ** | `lora_viz_utils.py` | 単一レイヤーの抽出、SVD計算、ヒートマップ描画などの基礎ロジック。 |
| **レポート・エンジン** | `lora_matrix_report.py` | 20x20マトリクスのバッチ処理、HTMLダッシュボードの構築。 |

## 2. ワークフローの分離 (Notebooks)
用途に応じてノートブックを2種類に分けました。

- **`lora_heatmap_visualization.ipynb`**:
    - **用途**: インタラクティブな深掘り解析。
    - **特徴**: 軽量。特定のレイヤーやタスクペア（例: Task 18 vs 19）を詳しく見たい時に使用。
- **`lora_matrix_report_batch.ipynb`**:
    - **用途**: 全タスク（400ペア）のレポート自動生成。
    - **特徴**: 放置実行用。Phase 1（重い画像生成）と Phase 2（軽いHTML構築）の2段構成。

## 3. 主要な実装・決定事項
- **階層化保存**: `Dataset/Model/Run/matrix_report/{layer}_{type}/plots/` の構造で結果を整理。
- **絶対スケールの固定**: ヒートマップのカラーバーを **0°〜90°** に固定し、直交性の比較を容易化。
- **自動判定ロジック**: チェックポイントのキーから Rank と Task 数を自動抽出する機能を搭載。
- **インタラクティブUI**:
    - HTMLダッシュボード上でクリックすると画像が大きくポップアップ（Lightbox）する機能を実装。
    - 400枚の画像を `plots/` サブフォルダに集約。

## 4. 今後の拡張予定
- 学習ステップごとの「基底の移動（時系列変化）」の可視化。
- 加重コーダル距離などのベクトル化された評価指標の統合。
